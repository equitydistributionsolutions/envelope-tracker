❓ How it works — index vs tracker dashboard
✅
What updates automatically from Google Sheets: The client-facing tracker dashboard (each .html file) fetches live data from its linked Google Sheet every 30 seconds. The moment you update the sheet, the client sees the change within 30 seconds — no action needed from you.
🖊️
What you update manually here in the index: Client names, distribution names, URLs, completion %, and status. This index is your internal control panel — it does not connect to Google Sheets. Update it when you add a new distribution, go live with a URL, or close out a distribution.
🔗
How they link together: Each distribution in the index has a URL field. That URL points to the tracker dashboard HTML file hosted on GitHub. Clients only ever see the tracker dashboard — they never see this index.
📁
File structure in GitHub: Keep one folder per client (e.g. ahead/, rjw/). Each distribution is one HTML file inside that folder. The index lives at the root. See the repo map above for the live structure based on your current data.
♻️
New distribution: Duplicate tracker-dashboard.html, rename it (e.g. jun-15-2026.html), update the 6 CONFIG values at the top of the script, upload to the client's GitHub folder, then add it here in the index with its new URL.
